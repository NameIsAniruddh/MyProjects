SEQUENCE:

Three Ways able to use sequence;
1. in select statement.
2. in insert statement.
3. default (at the time of table creation itself).


create sequence item_seq 
MINVALUE 1
MAXVALUE 1000
START WITH 1
INCREMENT BY 1
NOCYCLE
CACHE 20;


IDENTITY COLUMN   --> we can generally say its looks like a advance sequence having more option compare to sequence like (generate on DEfaulut, generate on default on NULL etc.)
---------------
identity column is one of very useful for the identity the particular raw
auto-generates and insert a sequential value


Key Words are: 
 GENERATED ALWAYS 
 GENERATED BY DEFAULT 
 GENERATED BY DEFAULT ON NULL 
 START WITH initial_value 
 INCREMENT BY internval_value 
 CACHE 


-----GENERATED BY DEFAULT ON NULL------------
CREATE  TABLE identity_demo ( 
id NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY START WITH 12 INCREMENT BY 4, 
description VARCHAR2(100) not null 
); 


insert into identity_demo(description) values ('RAJAVEL');
insert into identity_demo(description) values ('SARATH');

Output:
12	RAJAVEL
16	SARATH

-----GENERATED BY DEFAULT ON NULL------------
CREATE  TABLE identity_demo ( 
id NUMBER GENERATED ALWAYS AS IDENTITY START WITH 10 INCREMENT BY 10, 
description VARCHAR2(100) not null 
); 


insert into identity_demo(id,description) values (12345,'RAJAVEL');
insert into identity_demo(id,description) values (NULL,'SARATH');


output:
12345	RAJAVEL
12	SARATH
