Sub-Query : 
 In Oracle, a sub-query is a query within a query. You can create sub-queries within your SQL statements.  
 A sub-query may occur in : 
	 A SELECT clause 
	
		SELECT ATTENDANCE_ID , 
		EMPLOYEE_ID    ,
		(SELECT FIRST_NAME||LAST_NAME EMP_NAME FROM EMPLOYEES 
		WHERE EMPLOYEES.EMPLOYEE_ID = EMPLOYEE_ATTENDANCE.EMPLOYEE_ID) EMPLOYEE_NAME,
		ATTENDANCE_DATE,
		IN_TIME        ,
		OUT_TIME       ,
		STATUS         ,
		REMARKS        
		  FROM EMPLOYEE_ATTENDANCE;
	
	
	 A FROM clause 
	
		SELECT A.EMPLOYEE_ID, A.EMPLOYEE_NAME
		  FROM (
			SELECT ATTENDANCE_ID , 
		EMPLOYEE_ID    ,
		(SELECT FIRST_NAME||LAST_NAME EMP_NAME FROM EMPLOYEES 
		WHERE EMPLOYEES.EMPLOYEE_ID = EMPLOYEE_ATTENDANCE.EMPLOYEE_ID) EMPLOYEE_NAME,
		ATTENDANCE_DATE,
		IN_TIME        ,
		OUT_TIME       ,
		STATUS         ,
		REMARKS        
		  FROM EMPLOYEE_ATTENDANCE
		  ) A, 
		  (SELECT * EMPLOYEE_ATTENDANCE WHERE YEAR = 2021)B
		WHERE A.EMP_ID = B.EMP_ID

	 A WHERE clause 



Types of Sub-Query: 
 Single row sub-query              
 Multiple row sub-query          
	: Returns zero or one row. 
	: Returns one or more rows. 
 Multiple column sub-queries : Returns one or more columns. 
 Correlated sub-queries       
     Correlated sub-queries are the one in which inner query or sub-query reference outer query. Outer query needs to be executed before inner query.  
	 One of the most common example of correlated sub-query is using keywords exits and not exits.  
	 An important point to note is that correlated sub-queries are slower queries and one should avoid it as much as possible.   
     Correlated Sub-queries are slower than non correlated sub-query and should be avoided in favor of SQL joins.     
 Non-Correlated sub-queries : 
	 In non correlated sub-query, inner query doesn't depend on outer query and can run as stand alone query. 
	 Sub-query used along-with IN or NOT IN SQL clause is good examples of Non-correlated sub-query in SQL. 
 Nested sub-queries                   
	: Sub-queries are placed within another sub-query. 
	
	
Single row sub-Query : - (Within the sub-query as only one row) 
		SELECT EMPLOYEE_ID, 
		SALARY 
		FROM EMPLOYEES 
		WHERE SALARY = 
		( 
		SELECT  MAX(SALARY) 
		FROM EMPLOYEES 
		); 
Multi-Row Sub-Query : - (Within the sub-query as more than one row) 
		SELECT EMPLOYEE_ID, 
		SALARY 
		FROM EMPLOYEES 
		WHERE SALARY IN 
		( 
		SELECT  SALARY 
		FROM EMPLOYEES 
		WHERE DEPARTMENT_ID = 300 
		); 
Multi-Column Sub-Query : 
		SELECT EMPLOYEE_ID, 
		SALARY 
		FROM EMPLOYEES 
		WHERE SALARY,DEPARTMENT_ID IN 
		( 
		SELECT  SALARY,DEPARTMENT_ID 
		FROM EMPLOYEES 
		WHERE DEPARTMENT_ID = 300 
		); 
Correlated Sub-Query : - (inner query or sub-query reference outer query) 
		SELECT *
		  FROM EMPLOYEE_ATTENDANCE A
		 WHERE EXISTS (SELECT 1 FROM EMPLOYEES B WHERE B.EMPLOYEE_ID = A.EMPLOYEE_ID)



		SELECT DEPARTMENT_ID, 
		EMPLOYEE_ID, 
		SALARY 
		FROM EMPLOYEES E_0 
		WHERE 1 =  
		( 
		SELECT  COUNT(DISTINCT SALARY) 
		FROM EMPLOYEES E_I 
		WHERE E_O.DEPARTMENT_ID = E_I.DEPARTMENT_ID 
		AND E_O.SALARY <=  E_I.SALARY); 
		

Non-Correlated Sub-Query : (Inner does not depends on outer query) 

		SELECT *
		  FROM EMPLOYEE_ATTENDANCE
		WHERE (EMPLOYEE_ID,DEPARTMENT_ID) IN 
		(SELECT EMPLOYEE_ID,DEPARTMENT_ID FROM EMPLOYEES WHERE SALARY >= 60000); 

		SELECT COMPANY 
		FROM Stock 
		WHERE LISTED_ON_EXCHANGE IN 
		(SELECT RIC 
		FROM Market 
		WHERE COUNTRY = 'United States' OR COUNTRY = 'INDIA'); 
		
Nested Sub-Query : A sub-query is nested when you are having a sub-query in the where or having clause of another sub-query. 
		Select * 
		From result 
		where rollno in (select rollno 
		from student 
		where courseid = (select courseid 
		from student 
		where rollno = 12));

Hierarchial query: 
SELECT LEVEL, 
             CONNECT_BY_ROOT  PARENT_ID, 
       CONNECT_BY_ISLEAF, 
       LTRIM(SYS_CONNECT_BY_PATH(PARENT_ID, '-'), '-'), 
       PARENT_NAME, 
       CONNECT_BY_ISCYCLE, 
       CHILD_NAME FROM HIR_TABLE START WITH PARENT_ID =1 CONNECT BY NOCYCLE 
PARENT_ID = PRIOR CHILD_ID;
